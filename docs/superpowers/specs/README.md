# PR design specs

Per [AGENTS.md](../../../AGENTS.md), every implementation PR must have a dedicated spec in this directory before coding starts.

## Naming

`YYYY-MM-DD-pr-<n>-<topic>-design.md`

Example: `2026-05-28-pr-1-security-baseline-design.md`

## Minimum sections

- Problem statement and scope boundaries
- Current behavior vs target behavior
- Technical approach and alternatives considered
- Risk/failure modes and mitigations
- Test strategy and validation commands
- Rollout/backout notes (if operational behavior changes)

Link the spec from the PR description. Update the spec if scope changes materially.
